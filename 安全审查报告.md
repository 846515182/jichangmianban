## 1. 仪表盘（管理员总览 vs 用户专属）

### 风险等级: 🟠 高危

### 代码定位

**后端文件**:
- `internal/server/handler/admin/dashboard.go`
- `internal/server/handler/user/dashboard.go`

**前端文件**:
- `src/views/admin/Dashboard.vue`
- `src/views/user/Dashboard.vue`

### 发现的问题

#### 1.1 性能高危：在线人数统计无缓存

**文件**: `internal/server/handler/admin/dashboard.go:77-85`

```go
func (h *DashboardHandler) getOnlineCount() (int64, error) {
    var count int64
    err := h.db.QueryRow(context.Background(), `
        SELECT COUNT(DISTINCT user_id) 
        FROM traffic_realtime 
        WHERE updated_at > NOW() - INTERVAL '5 minutes'
    `).Scan(&count)
    return count, err
}
```

**问题**: 每次请求都实时查询数据库，无 Redis 缓存。高并发场景下会导致数据库压力激增。

**攻击复现 Payload**:
```http
GET /api/v1/admin/dashboard/online-count HTTP/1.1
Host: panel.example.com
Authorization: Bearer admin_token

# 并发 1000 次请求，数据库 QPS 激增
```

**修复建议**:
```go
// 使用 Redis 缓存，5分钟过期
func (h *DashboardHandler) getOnlineCount() (int64, error) {
    cacheKey := "dashboard:online_count"
    if val, err := h.redis.Get(cacheKey); err == nil {
        return strconv.ParseInt(val, 10, 64)
    }
    // 查询数据库并缓存
    count := queryDB()
    h.redis.Set(cacheKey, count, 5*time.Minute)
    return count, nil
}
```

#### 1.2 横向越权风险：用户仪表盘参数篡改

**文件**: `internal/server/handler/user/dashboard.go:23-35`

```go
func (h *DashboardHandler) getStats(r *http.Request) (*Stats, error) {
    userID := getUserID(r) // 从 context 获取，安全 ✓
    var stats Stats
    err := h.db.QueryRow(context.Background(), `
        SELECT 
            COALESCE(SUM(total_upload + total_download), 0) as total_traffic,
            COALESCE(SUM(total_upload), 0) as total_upload,
            COALESCE(SUM(total_download), 0) as total_download
        FROM traffic_logs 
        WHERE user_id = $1
    `, userID).Scan(&stats.TotalTraffic, &stats.TotalUpload, &stats.TotalDownload)
    return &stats, err
}
```

**问题**: 虽然当前代码使用 `getUserID(r)` 从 context 获取用户 ID（安全），但需要验证 `getUserID()` 的实现是否真正从 JWT token 解析，而非从请求参数。

**验证方法**:
```http
GET /api/v1/user/dashboard/stats?user_id=2 HTTP/1.1
Host: panel.example.com
Authorization: Bearer user1_token

# 如果返回 user_id=2 的数据，则存在横向越权
```

**修复建议**:
```go
// 确保 getUserID 从 JWT 解析，忽略请求参数
func getUserID(r *http.Request) int64 {
    claims := r.Context().Value("claims").(*jwt.Claims)
    return claims.UserID // 严禁从 r.URL.Query().Get("user_id") 获取
}
```

#### 2.1 敏感凭证泄露：节点密码明文存储

**文件**: `internal/server/handler/admin/node.go:45-62`

```go
func (h *NodeHandler) createNode(r *http.Request) error {

    }

        VALUES ($1, $2, $3, $4, $5, $6)
    `, req.Name, req.Address, req.Port, req.Password, req.Method, req.TrafficLimit)
    // req.Password 明文直接插入数据库 🔴
    return err
}
```

**问题**: 节点 `password` 字段以明文形式存储在数据库中，一旦数据库泄露，所有节点密码暴露。

**攻击复现 Payload**:
```http
POST /api/v1/admin/nodes HTTP/1.1
Host: panel.example.com
Authorization: Bearer admin_token
Content-Type: application/json

{
  "name": "Test Node",
  "address": "1.2.3.4",
  "port": 8388,
  "password": "MySecretPassword123",  // 明文存储
  "method": "aes-256-gcm",
  "traffic_limit": 107374182400
}

# 查询数据库验证
SELECT password FROM nodes WHERE name = 'Test Node';
# 返回: MySecretPassword123 (明文)
```

**修复建议**:
```go
// 使用 AES-GCM 加密存储
func encryptPassword(plaintext string) (string, error) {
    key := os.Getenv("ENCRYPTION_KEY") // 32字节密钥
    block, _ := aes.NewCipher([]byte(key))
    gcm, _ := cipher.NewGCM(block)
    nonce := make([]byte, gcm.NonceSize())
    io.ReadFull(rand.Reader, nonce)
    ciphertext := gcm.Seal(nonce, nonce, []byte(plaintext), nil)
    return base64.StdEncoding.EncodeToString(ciphertext), nil
}

// 创建节点时加密
encryptedPwd, _ := encryptPassword(req.Password)
h.db.Exec(..., encryptedPwd)
```

#### 2.3 配置下发安全：需验证 token 签名

**文件**: `internal/server/handler/user/node.go:67-89`

```go
func (h *NodeHandler) getConfig(nodeID int64, r *http.Request) (*NodeConfig, error) {
    userID := getUserID(r)
    
    // 验证用户是否有权访问该节点
    var hasAccess bool
    err := h.db.QueryRow(context.Background(), `
        SELECT EXISTS(
            SELECT 1 FROM node_plan_bindings npb
            JOIN subscriptions s ON s.plan_id = npb.plan_id
            WHERE npb.node_id = $1 AND s.user_id = $2 AND s.status = 'active'
        )
    `, nodeID, userID).Scan(&hasAccess)
    
    if !hasAccess {
        return nil, errors.New("no access")
    }
    
    // 生成配置
    config := generateConfig(nodeID, userID)
    return config, nil
}
```

**问题**: 需要检查 `generateConfig()` 函数生成的 token 是否包含用户身份签名，防止订阅链接被盗用。

**验证方法**:
```http
GET /api/v1/user/nodes/1/config HTTP/1.1
Host: panel.example.com
Authorization: Bearer user1_token

# 返回的 config 中 token 字段应包含用户签名
# 如果 token 可被其他用户使用，则存在漏洞
```

**修复建议**:
```go
func generateConfig(nodeID, userID int64) *NodeConfig {

        "node_id": nodeID,
        "exp":     time.Now().Add(24 * time.Hour).Unix(),
    })
    tokenString, _ := token.SignedString([]byte(os.Getenv("JWT_SECRET")))
    
    return &NodeConfig{
        Token: tokenString,
        // ...
    }
}
```

#### 4.1 订阅链接唯一性：需验证 token 生成方式

**文件**: `internal/server/handler/user/subscription.go:45-62`

```go
func (h *SubscriptionHandler) subscribe(r *http.Request) error {
    userID := getUserID(r)
    var req SubscribeRequest
    json.NewDecoder(r.Body).Decode(&req)
    
    // 生成订阅 token
    token := generateSubscribeToken(userID, req.PlanID)

        VALUES ($1, $2, $3, 'active')

    return err

```

**问题**: 需要检查 `generateSubscribeToken()` 函数是否使用强随机数生成器。如果使用 `md5(time()+rand)` 则存在被暴力猜解风险。

**验证方法**:
```go
// 检查 generateSubscribeToken 实现
func generateSubscribeToken(userID, planID int64) string {
    // 如果是这样，则存在漏洞 🔴
    return fmt.Sprintf("%x", md5.Sum([]byte(fmt.Sprintf("%d%d", time.Now().Unix(), rand.Int()))))
    
    // 应该是这样 ✓
    b := make([]byte, 32)
    rand.Read(b)
    return hex.EncodeToString(b)
}
```

**修复建议**:
```go
func generateSubscribeToken(userID, planID int64) string {
    b := make([]byte, 32)
    _, _ = rand.Read(b) // 使用 crypto/rand
    return hex.EncodeToString(b)
}
```

#### 6.1 支付回调幂等性缺失

**文件**: `internal/server/handler/user/order.go:123-145`

```go
func (h *OrderHandler) handlePaymentCallback(r *http.Request) error {
    var req PaymentCallbackRequest
    json.NewDecoder(r.Body).Decode(&req)
    
    // 仅判断 trade_status 成功，未校验 out_trade_no 是否已处理 🔴
    if req.TradeStatus == "SUCCESS" {
        _, err := h.db.Exec(context.Background(), `
            UPDATE orders 
            SET status = 'paid', paid_at = NOW()
            WHERE order_no = $1
        `, req.OutTradeNo)
        
        // 直接增加用户余额/流量，无状态锁
        h.db.Exec("UPDATE users SET balance = balance + $1 WHERE id = $2", req.Amount, req.UserID)
    }
    return err
}
```

**问题**: 缺少 `is_paid` 状态锁，黑客可重放回调包无限刷钱。

**攻击复现 Payload**:
```http
POST /api/v1/payment/alipay/callback HTTP/1.1
Host: panel.example.com
Content-Type: application/json

{
  "out_trade_no": "ORDER_123456",
  "trade_status": "SUCCESS",
  "amount": 100.00,
  "user_id": 1
}

# 重放 100 次
for i in {1..100}; do
  curl -X POST /api/v1/payment/alipay/callback -d '{...}'
done

# 用户余额增加 10000 元
```

**修复建议**:
```go
func (h *OrderHandler) handlePaymentCallback(r *http.Request) error {
    // 使用事务 + 行锁
    tx, _ := h.db.Begin(context.Background())
    
    var order Order
    tx.QueryRow("SELECT id, status FROM orders WHERE order_no = $1 FOR UPDATE", req.OutTradeNo).Scan(&order)
    
    if order.Status == "paid" {
        tx.Rollback()
        return errors.New("already paid")
    }
    
    tx.Exec("UPDATE orders SET status = 'paid' WHERE id = $1", order.ID)
    tx.Exec("UPDATE users SET balance = balance + $1 WHERE id = $2", req.Amount, req.UserID)
    
    tx.Commit()
    return nil
}
```

```go
func (h *CouponHandler) useCoupon(r *http.Request) error {
```
